# Git Log

## Definition

`git log` is a Git command used to display the commit history of a repository. It shows all commits in reverse chronological order (latest commit first).

---

## Syntax

```bash
git log
```

```bash
git log --oneline
```

---

## Why do we need git log?

- To view the complete commit history.
- To identify a specific commit using its Commit Hash.
- To know who made the commit.
- To know when the commit was made.
- To view the commit message.
- To understand the project history.

---

## What is a Commit?

A commit is a snapshot of the project at a specific point in time. Every commit stores the current state of the project.

A commit contains:

- Project Snapshot
- Commit Hash
- Author
- Date & Time
- Commit Message
- Parent Commit Reference

---

## Commit Hash

A Commit Hash is a unique identifier automatically generated by Git for every commit.

Example:

```
1e39e1d3dd4fa10b1b2ac1d9872541d73d6f7086
```

No two commits have the same hash.

It is used to uniquely identify a commit.

---

## Output of git log

Example:

```
commit 1e39e1d3dd4fa10b1b2ac1d9872541d73d6f7086
Author: Ajit Kale <ajitkale84@gmail.com>
Date: Tue Jul 14 16:52:16 2026 +0530

    Test commit
```

### Explanation

**Commit**
- Indicates the start of a commit.

**Commit Hash**
- Unique ID of the commit.

**Author**
- User who created the commit.

**Date**
- Date and time when the commit was created.

**Commit Message**
- Description provided while executing `git commit -m`.

---

## Order of Commits

Git displays commits in **reverse chronological order**.

Latest Commit
↓

Older Commit
↓

Oldest Commit

---

# HEAD

HEAD is a pointer that always points to the latest commit of the currently checked-out branch.

Example:

```
HEAD
 │
 ▼
main
 │
 ▼
Commit 5
```

Whenever a new commit is created, both **HEAD** and the **current local branch** move to the latest commit.

---

## Local Branch (main)

`main` is the current local branch in the Local Repository.

It always points to the latest commit of that local branch.

---

## Remote Tracking Branch (origin/main)

`origin/main` is the remote tracking branch.

It represents the latest commit available on the remote repository (GitHub or Azure DevOps).

It moves only after executing:

```bash
git push
```

---

## Pointer Movement

### After git commit

Moves:
- HEAD
- Current Local Branch (main)

Does NOT Move:
- origin/main

### After git push

Moves:
- origin/main

---

## Key Points

- `git log` displays commit history.
- Latest commit is displayed first.
- Every commit has a unique Commit Hash.
- HEAD always points to the latest commit of the current branch.
- `main` is the local branch.
- `origin/main` represents the remote branch.
- `git commit` moves HEAD and the local branch.
- `git push` updates the remote tracking branch.
