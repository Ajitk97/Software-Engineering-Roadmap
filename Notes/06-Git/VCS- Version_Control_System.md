# VCS- Version Control System
--------------------

## What is a Version Control System (VCS)?

Definition

A Version Control System (VCS) is a software tool that helps developers track changes made to files over time. It allows us to save different versions of a project, restore previous versions, and collaborate with multiple developers without losing work.

## Why do we need a VCS?

Imagine you're writing your Git notes.

- Day 1

You write:

Git Basics

Save the file.

- Day 2

You add:

Git Commands

Save the file.

- Day 3

You accidentally delete the entire document and save it.

Without a Version Control System:

❌ Your previous work is lost unless you have a backup.

With a Version Control System:

✅ You can restore an earlier version of the file.

## Problems Solved by a VCS

- Without a VCS:

 >Difficult to restore older versions.
Easy to overwrite someone else's work.
No record of who changed what.
Hard to collaborate in teams.

- With a VCS:

Complete history of changes.
Easy to restore previous versions.
Team collaboration becomes simpler.
Every change is recorded.
